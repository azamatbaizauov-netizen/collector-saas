Дневная кластеризация ленты рабочей WhatsApp-группы «чеки» оптовика (ADR 0008): сколько РАЗНЫХ людей вернуло долг за сутки и на какую сумму.

## Контекст
В группу за день падают чеки оплаты (в основном Kaspi, тенге), скрины курса обмена и текст «такой-то вернул такую-то сумму». Нужен не подсчёт «по числу чеков», а рассуждение по смыслу: несколько чеков + скрин курса от одного менеджера по одному клиенту — это ОДИН возврат; текст «Иван вернул 500» — отдельное событие. Итог — информационная сводка владельцу (Медету) в 18:00, долг в листе не меняем (принцип 1, ADR 0003).

## Входные данные
- transcript: весь дневной поток группы по порядку. Каждая строка начинается с префикса «Менеджер X:» (кто запостил — сопоставляется в коде по номеру отправителя), далее текст и/или строка разбора чека vision'ом (docType/сумма/валюта/плательщик/…).

## Выход
JSON: `{"payers":[{"name":"Иван","amount":500,"currency":"USD","manager":"Адильбек"},{"name":"Альфа ТОО","amount":250000,"currency":"KZT","manager":"Айнур"}]}`
- payers — массив установленных возвратов (по человеку). Пусто, если возвратов нет.
- name — кто вернул (имя плательщика с чека или из текста; если не понять — краткое описание).
- amount — сумма ЧИСЛОМ в валюте платежа. НЕ конвертируем.
- currency — ISO: KZT | USD | RUB | CNY.
- manager — менеджер из префикса «Менеджер X:» строки, где виден платёж («—», если не сопоставлен). Разбивка сводки по менеджерам — в коде.

## Правила
- Кластеризуй по смыслу: несколько чеков + скрин курса по одному клиенту = один возврат.
- Скрин курса сам по себе — не платёж.
- Один платёж, видимый и чеком, и текстом, — не удваиваем.
- Фото товара / болтовня / приветствия — игнор.
- Сомневаешься, платёж это или нет — не включаем.
- Валюту НЕ конвертируем (Kaspi-чек = KZT). Суммирование по валютам делает код, не модель.
- Ответ — только JSON, без пояснений.
